你是一位熟悉 AI Agent 工作流程的技術文件撰寫者。
請分析這個 Repo 的結構與內容，產生一份高品質的 `AGENTS.md`。

## 核心原則

### 原則一：漸進式揭露 — Map, not Manual（地圖，不是手冊）

AGENTS.md 是導航地圖，不是操作手冊。

**正確用途**：告訴 Agent「去哪裡找什麼」，詳細內容放在連結的文檔裡。

```
AGENTS.md（地圖）
 → docs/architecture.md        分層架構詳細說明
 → docs/development.md         開發環境搭建
 → docs/design-docs/ref-*.md   參考專案架構說明
 → docs/design-docs/*-patterns.md  元件使用模式
```

**禁止**：
- 在 AGENTS.md 中複製貼上其他文檔的詳細內容
- 逐行說明實作細節
- 列出所有設定值或完整程式碼

**判斷準則**：每個段落都應能回答「這讓 Agent 知道去哪找資訊」，而非「這直接給 Agent 所有資訊」。

---

### 原則二：以 Agent 行為為中心

內容聚焦在「Agent 做什麼任務時需要知道什麼」，而非「這個 Repo 是什麼」。

優先說明：
- Agent 在此 Repo 執行任務的入口點
- 哪些指令、技能、規範需要遵守
- 哪些文件在哪些任務情境下應該被參考

---

### 原則三：簡短精煉

- 每個連結一行，附上一句話描述用途
- 不寫廢話，不重複，不解釋顯而易見的事
- 讀完整份文件不超過 2 分鐘

---

## 輸出格式要求

請依以下結構產生 AGENTS.md：

```markdown
# AGENTS.md

> 一句話說明這個 Repo 的用途與 Agent 的角色。

## 快速導航

| 情境 | 參考文件 |
|------|----------|
| 任務情境一 | [文件名稱](路徑) |
| 任務情境二 | [文件名稱](路徑) |

## 核心規範

- [規範名稱](路徑) — 一句話說明何時使用

## 技能與工具

- [技能名稱](路徑) — 一句話說明用途

## 重要約束

列出 2-5 條 Agent 必須遵守的硬性規則（不超過一行）。
```

## 執行步驟

1. 掃描 Repo 的頂層結構與 `README.md`（若存在）
2. 找出 `docs/`、`instructions/`、`skills/`、`patterns/`、`prompts/` 等關鍵目錄
3. 依照「Map, not Manual」原則，只列出導航連結，不複製內容
4. 依上述格式輸出 `AGENTS.md`，**不得包含任何詳細實作說明**
5. 最後確認：每個段落是否都只是「指路」而非「給答案」
