你是一位熟悉 AI Agent 工作流程的技術文件撰寫者。
請分析這個 Repo 的結構與內容，產生一份高品質的 `AGENTS.md`。

## 核心原則

### 原則一：漸進式揭露 — Map, not Manual（地圖，不是手冊）

AGENTS.md 是導航地圖，不是操作手冊。

**正確用途**：告訴 Agent「去哪裡找什麼」，詳細內容放在連結的文檔裡。

```
AGENTS.md（地圖）
 → docs/architecture.md        分層架構詳細說明
 → docs/development.md         開發環境搭建
 → docs/design-docs/ref-*.md   參考專案架構說明
 → docs/design-docs/*-patterns.md  元件使用模式
```

**禁止**：

- 在 AGENTS.md 中複製貼上其他文檔的詳細內容
- 逐行說明實作細節
- 列出所有設定值或完整程式碼

**判斷準則**：每個段落都應能回答「這讓 Agent 知道去哪找資訊」，而非「這直接給 Agent 所有資訊」。

---

### 原則二：以 Agent 行為為中心

內容聚焦在「Agent 做什麼任務時需要知道什麼」，而非「這個 Repo 是什麼」。

優先說明：

- Agent 在此 Repo 執行任務的入口點
- 哪些指令、技能、規範需要遵守
- 哪些文件在哪些任務情境下應該被參考

---

### 原則三：簡短精煉

- 每個連結一行，附上一句話描述用途
- 不寫廢話，不重複，不解釋顯而易見的事
- 讀完整份文件不超過 2 分鐘

---

## 輸出格式要求

請依以下結構產生 AGENTS.md。**每個 section 只寫摘要與連結，詳細內容留給連結的文件**：

```markdown
# AGENTS.md

## 1. 專案概述

一段話說清楚：專案是什麼、技術堆疊、倉庫結構（前 10 行讓 Agent 建立心智模型）。

## 2. 快速指令

建置、啟動、格式化、品質檢查的命令速查表。
環境變數配置說明（env 檔案位置、啟動腳本）。

## 3. 後端架構

包結構樹（ASCII）+ 每個包的一行用途註釋。
核心子系統簡述 → [詳細說明](路徑)

## 4. 前端架構

技術棧、路由方案、API 層約定、元件庫規範。
→ [詳細說明](路徑)

## 5. 關鍵約定

5–10 條硬性規則（違反會直接導致問題的）。
每條規則附連結 → [詳細說明](路徑)

## 6. 本機開發及驗證流程

「改 → 建置 → 啟動 → 驗證」完整閉環。
curl 驗證範本、Token 取得方式、日誌路徑。

## 7. 品質檢查

lint / format / build / test 指令矩陣（表格形式）。

## 8. 參考項目約定

參考項目清單 + 優先順序規則。
→ [詳細說明](路徑)

## 9. 文件導航

| 文件             | 用途       |
| ---------------- | ---------- |
| [文件名稱](路徑) | 一句話說明 |
```

## 執行步驟

1. 掃描 Repo 頂層結構與 `README.md`（若存在），建立整體心智模型
2. 找出 `docs/`、`instructions/`、`skills/`、`patterns/`、`prompts/` 等關鍵目錄
3. 依照「Map, not Manual」原則，每個 section 只寫摘要與連結，不複製詳細內容
4. 若某個 section 對應的詳細文件不存在，只寫一行摘要，**不展開細節**
5. 依上述格式輸出 `AGENTS.md`
6. 最後自我檢查：每個段落是否都只是「指路」而非「給答案」
