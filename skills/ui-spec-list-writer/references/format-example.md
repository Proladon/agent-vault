# 格式範例

以下為輸出格式的結構解析，對應真實案例。

---

## 基本結構

```
- {change-type} - {頁面或元件名稱}（附加說明）
  - {功能區塊}
    - `{元件名稱}` 欄位旁新增 `{新增項目}` 按鈕
      - 點擊後開啟「{Drawer 名稱}」
    - 新增 "{區塊名稱}" 區塊
      - 區塊標題: `{標題文字}`，右側顯示 `{按鈕文字}` 按鈕（點擊行為待確認）
      - 內容以 **輪播 (carousel)** 方式呈現，底部有分頁圓點
      - 每張卡片顯示內容：
        - **{子區域名稱}**（位置說明，如「左上角，兩個 tag 並排」）
          - 條件 1：顯示 `{文字}`
          - 條件 2：顯示 `{文字}`
        - **{標題行}**（位置說明）
          - 左側: `{內容格式}`
          - 右側: `{內容格式}`（例：`NT$ {累計金額} / {目標金額}`）
        - **{進度條}**（說明）
        - **{說明文字}**（位置說明）
          - 格式 A: `{條件類型} {格式說明}`
          - 格式 B: `{條件類型} {格式說明}`

- new - {新功能名稱} drawer
  - 觸發來源：{觸發頁面} 的「{按鈕名稱}」按鈕 / {其他觸發來源}
  - **Drawer 標題**：`{標題文字}`
  - **頂部資訊列**（{欄數}欄）
    - `{欄位名稱}`：{說明}
    - `{欄位名稱}`：{說明}（{特殊情況}則顯示 `{替代文字}`）
  - **分頁 tabs**：`{tab1}` | `{tab2}`
    - 預設顯示 `{預設 tab}`
  - **{內容區塊名稱}**（依分頁顯示對應內容）
    - {區塊內的排列或規則說明}
    - **{子區塊標題列}**
      - 左側：{說明}
      - 右側：
        - {條件 A} → 顯示 `{文字}`
        - {條件 B} → 顯示 `{文字}` + `{附加資訊}`
    - **{卡片名稱}**（每個目標下可有多張）
      - **{區域名稱}**（卡片位置）
        - **{tag 1}**：`{值 A}` 或 `{值 B}`
        - **{tag 2}**（僅{條件}模式下顯示）
          - `{狀態值 A}`：{描述}
          - `{狀態值 B}`：{描述}
      - **{標題行}**
        - 左側：`{格式}`
        - 右側：{格式說明}
      - **{進度條}**
      - **{說明文字}**
        - {情境 A}：`{格式}`
        - {情境 B}：`{格式}`
      - **{區間欄位}**：`{格式}`
      - **{自訂義區域}**（說明：{無內容時的處理}）
```

---

## Tooltip 寫法

```
- 新增 tooltip
  - 顯示條件: 等級 1 ~ 4 的「設定有效期限」都加上 Tooltip
  - 內容: 等級到期時，系統判斷會員是否符合續等條件；永久則不會判斷
```

---

## 按鈕改名寫法

```
- 原 `查看` 按鈕改名為 `等級異動紀錄`
- 新增 `等級進度條` 按鈕
  - 點擊後開啟「等級進度條 drawer」
```

---

## Tab 拆分寫法

```
- {點數名稱} 紀錄 tab
  - 點數使用紀錄 table 改為兩個 tab (`點數餘額明細` & `點數使用紀錄`)
```

---

## 新增選項寫法

```
- 新增選項: `有效期 {input} 月`
  - tooltip: 當第一等級到期時，會依照等級一的續等後設定決定消費認列起始日是否更新
```
