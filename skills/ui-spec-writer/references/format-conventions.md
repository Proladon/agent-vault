# Format Conventions

## 文件結構規則

### 開頭段落（無標題）

第一段直接描述這次「做了什麼異動」，例如：

> 在會員模組底下新增等級禮設定頁面類別，子頁面有 "入會禮設定", "升等禮設定", "等級獨享禮設定"

### 標題層級

- `###` → 主頁面（列表頁、設定頁、紀錄頁）
- `####` → 子頁面（編輯頁、Drawer、Modal）

---

## 頁面描述規則

### 列表 / 設定頁

先用一句話說明頁面的呈現方式，再畫 ASCII：

> 以設定區塊的方式條列呈現各個等級，每個等級區塊都有編輯按鈕，點擊「編輯」按鈕後前往「編輯 {名稱} XXX設定」頁

### 編輯頁 / 表單

先說明起始狀態（有哪些固定區塊），再依序列出欄位：

> 起初會有「發放設定」區塊，區塊內有:

### 相似頁面

直接引用已描述的頁面，只描述差異：

> 與 `入會禮設定頁` 形式一致，但在每個等級區塊中的「啟用狀態」下多一個「領取次數」

## ASCII 示意圖規則

### 卡片 / 設定區塊

```
+---------------------------------------------------------------------------------+
|                                                                                 |
|  | {標題}                                                       [ 按鈕 ]  |
|                                                                                 |
|   欄位標簽            欄位值                                                    |
|                                                                                 |
|   欄位標簽            值1                                                       |
|                       值2（多行值）                                             |
|                                                                                 |
+---------------------------------------------------------------------------------+
```

### 過濾器佈局

用簡單的文字排列表示欄位在同一行或換行：

```
欄位A
欄位B  欄位C
欄位D與欄位E
```

### 列表行欄位

不需要畫完整表格，直接用文字列出欄位名稱即可。

### 佔位變數

使用 `{變數名稱}` 表示動態內容，例如 `{堂票名稱}`、`{數量}`

## 欄位標注規則

### Component 標注格式

欄位後面接括號，寫明 component 類型與限制：

| 標注                                             | 說明                            |
| ------------------------------------------------ | ------------------------------- |
| `(開關)`                                         | Toggle switch                   |
| `(下拉選單, 多選, 必填)`                         | Multi-select dropdown, required |
| `(下拉選單, 單選)`                               | Single-select dropdown          |
| `(數字輸入 + {單位})`                            | Number input with unit suffix   |
| `(數字輸入 + 元)`                                | Number input with 元 suffix     |
| `(BaseDataSelect, 單選, placeholder: 請選擇XXX)` | 標準 BaseDataSelect component   |
| `(MemberSearch 組件)`                            | 會員搜尋組件                    |
| `(堂票進階選擇器)`                               | 進階選擇器                      |
| `(票券進階選擇器)`                               | 進階選擇器                      |
| `(圖文訊息組件)`                                 | 圖文訊息 component              |
| `(daterange picker)`                             | 日期範圍選擇器                  |
| `(tag)`                                          | Status tag，後面接狀態清單      |

### placeholder 寫法

必填欄位加 placeholder 標注：

> 發放種類(下拉選單, 多選, 必填)  
> placeholder: 請選擇發放種類

### 動態新增欄位組

> 新增發放堂票按鈕（多新增「發放堂票」、「發放數量」區塊，第二組開始都會有移除按鈕）

## 條件顯示規則

### 空值邊界

欄位有空值情況要在 ASCII 或緊接 ASCII 後說明：

> 發放內容 如果還沒有設定的話顯示「尚未設定」

### 條件欄位

某些欄位只在特定條件出現，用括號內的「註：」說明：

> - 回收數量（註：發送狀態為「已回收」或「回收失敗」才會有此欄位）

### 條件觸發的子區塊

> 選了哪些種類選項，底下就會出現對應的種類設定區塊

---

## Tag 狀態規則

列出所有狀態與對應 tag 類型：

> 狀態: 待處理(warning), 發放中(warning), 已發放(action), 部分失敗(danger), 發放失敗(danger)

常見 tag 類型：`warning`, `action`, `danger`, `info`

條件型 tag（根據數值決定）可用條件式描述：

> - **已回收**：發放數量 ≤ 回收數量
> - **回收失敗**：回收數量 = 0

---

## 時間格式規範

- 完整時間：`yyyy/mm/dd HH:mm:ss`
- 省略秒：`yyyy/mm/dd HH:mm`
- 可能無值：`yyyy/mm/dd HH:mm 或 "-"`

---

## Drawer 規則

Drawer 用 `####` 標題，命名格式：`drawer: {Drawer名稱}`

結構：

1. 上方基本資訊欄位（用 `-` 列表）
2. 區塊標題（例如：區塊：XXX明細）
3. 卡片內欄位（用 `-` 列表，加必要的「註：」條件說明）
4. 底部操作區（按鈕列表）
